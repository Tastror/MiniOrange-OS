# 依赖规范

## 文件夹具体依赖信息

如有缺陷，欢迎调整。

```plaintext
"include/common" 用于仅和 C 相关的易用的定义、函数等（如 u32、assert）
只能依赖
    "include/common"
无 .c 文件


"include/software_define" 用于与设备无关的量的定义（如文件系统、内核信息等）
只能依赖
    "include/common",
    "include/software_define"
无 .c 文件


"include/hardware_define" 用于与设备相关的量的定义（如 CPU 汇编信息等）
能依赖
    "include/common",
    "include/hardware_define"
无 .c 文件


--- 后面的内容里，都不可以再出现泛用且非宏函数的宏定义 ---


"include/lib" 用于和 C 相关的有用的函数，如 memset 等；以及朴素的对显示器的输出函数 printf 等（否则直到 kern 写完都没法输出，这是不好的）
能依赖
    "include/common",
    "include/software_define", "include/hardware_define",
    "include/lib"


"include/device" 用于设备驱动（如 pci、网卡等）
能依赖
    "include/common",
    "include/software_define", "include/hardware_define",
    "include/lib",
    "include/device"


"include/kernel" 用于内核相关内容
能依赖
    "include/common",
    "include/software_define", "include/hardware_define",
    "include/lib",
    "include/device",
    "include/kernel"
```

kernel 这里应该继续细分，暂且作如此。
